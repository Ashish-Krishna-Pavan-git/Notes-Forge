STARTER-FIRST RULES:
1. Prefer this starter marker set unless the request clearly needs something else:
   H1 H2 H3 H4 H5 H6
   BODY PARAGRAPH PARA
   BULLET NUMBERED
   CODE TABLE IMAGE FIGURE DIAGRAM HIGHLIGHT
   CAPTION
   TOC LOT LOF
   CHAPTER REFERENCES REFERENCE APPENDIX
   PAGEBREAK
2. Start every new block with `MARKER: payload`.
3. After a marker starts a block, keep every following non-marker line in that same block until the next marker appears.
4. This sticky-block rule is especially important for H1-H6, BODY, CAPTION, REFERENCE, CODE, and DIAGRAM.
5. Preserve indentation and tabs exactly inside CODE and DIAGRAM blocks.
6. Use `CAPTION:` immediately after a TABLE, IMAGE, FIGURE, or DIAGRAM when a caption is needed.
7. Use `NUMBERED:` for automatic numbering. Do not waste tokens on manual 1/2/3 prefixes unless the numbering text itself matters.
8. Keep wording concise, professional, and deterministic.
9. Return ONLY marker lines. No markdown fences, comments, prose, or JSON.

OPTIONAL ADVANCED MARKERS:

PAYLOAD REMINDERS:
- IMAGE / FIGURE: source | caption | align | scale
- TABLE: first row is the header
- HIGHLIGHT: text | color
- CODE / DIAGRAM: preserve indentation and tabs
- TOC / LOT / LOF / REFERENCES / PAGEBREAK can be marker-only lines
