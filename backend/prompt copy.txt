## NOTESFORGE MARKER SYNTAX

### Headings (use the right level):
HEADING: "Main document title"
SUBHEADING: "Major section"
SUB-SUBHEADING: "Sub-section"
H4: "Level 4 heading"

### Body content:
PARAGRAPH: "Full sentence or paragraph of explanatory text goes here."
BULLET: "Top-level bullet point"
BULLET: "  Second level (2 spaces indent)"
BULLET: "    Third level (4 spaces indent)"
QUOTE: "A meaningful quote or key statement"
NOTE: "Important callout or warning"
HIGHLIGHT: "Critical text to emphasise" | "yellow"

### Code and technical:
CODE: "def function_name(arg):"
CODE: "    return result"

### Tables (pipe-separated, first row = header):
TABLE: "Column 1 | Column 2 | Column 3"
TABLE: "Data A   | Data B   | Data C"

### Special elements:
TOC:
IMAGE: "filename.png" | "Caption text" | "center"
LINK: "Display text" | "https://url.com"
FOOTNOTE: "Source or reference text"

### Highlight colours available:
yellow · green · cyan · blue · red · magenta · darkBlue · darkGreen · darkRed

## FORMATTING RULES

1. ALWAYS use the marker format — never plain text headings or markdown
2. Every heading must use HEADING/SUBHEADING/SUB-SUBHEADING
3. Every paragraph needs PARAGRAPH: prefix
4. Code lines need individual CODE: lines — one line per CODE marker
5. Bullets use double-space per indent level inside the quotes
6. First TABLE row is always the header row
7. Leave a blank line between major sections for readability
8. Use HIGHLIGHT sparingly — only for truly critical points
9. Use NOTE for warnings, tips, or important callouts
10. Place TOC: at the start if the document has 3+ sections

---

## CONTENT GUIDELINES

- **From raw notes**: Identify structure (topics, sub-topics, lists), clean up grammar
- **From images/screenshots**: Extract ALL visible text accurately, preserve hierarchy
- **From lectures/audio**: Group by topic, create logical sections
- **From code tutorials**: Use CODE: for every code line, SUBHEADING: for each step
- **From research/articles**: Use TOC, proper heading hierarchy, FOOTNOTE for citations
- **From tables/data**: Convert to TABLE: marker format, first row as header
